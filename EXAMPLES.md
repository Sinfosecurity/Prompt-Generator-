## Example 1: Invoice Generator

**Inputs Used:**
- App Type: Invoice generator
- Target Users: Freelancers or solopreneurs
- Problem: Users struggle to create professional invoices quickly without design skills or complex software
- Inputs: Text or descriptions, Numbers or values, Multiple form fields
- Outputs: Generated text, A summary or report-style result, Clear "what to do next" guidance
- Color Scheme: Blue and white, professional
- Visual Style: Clean
- Tone: Professional
- Platform: Both desktop and mobile

**Generated Prompt (Universal):**

```
# AI-Ready Web Application Prompt

This prompt is designed to work with any AI builder (Cursor, Antigravity, v0, etc.).
It contains all necessary information to build the application without follow-up questions.

---

## 1. APPLICATION PURPOSE

**App Type:** Invoice Generator

**Primary Goal:** Users struggle to create professional invoices quickly without design skills or complex software

**Target Users:** Freelancers or solopreneurs

**Success Criteria:**
- Users can complete their task with minimal friction
- The interface is intuitive and requires no technical knowledge
- Results are delivered clearly and actionably
- The app handles edge cases gracefully

---

## 2. INPUT REQUIREMENTS

**User will provide:**
- Text or descriptions
- Numbers or values
- Multiple form fields

**Input Validation:**
- All required fields must be validated before processing
- Provide clear, helpful error messages for invalid inputs
- Use appropriate input types (text, number, file upload, etc.)
- Show inline validation feedback as users type
- Never accept empty or malformed inputs

---

## 3. OUTPUT SPECIFICATION

**After submission, display:**
- Generated text (emails, prompts, content)
- A summary or report-style result
- Clear "what to do next" guidance

**Output Requirements:**
- Results must be clear, scannable, and actionable
- Use visual hierarchy to emphasize key information
- Provide a "what to do next" section
- Include a way to copy, download, or share results
- If generating text, use proper formatting (headings, lists, spacing)

---

## 4. USER FLOW

1. **Landing:** User sees clear headline explaining what the app does
2. **Input:** User provides required information through well-labeled form fields
3. **Validation:** System validates inputs and shows helpful error messages if needed
4. **Processing:** Clear loading state while processing (if applicable)
5. **Results:** Display output in organized, readable format
6. **Action:** User can copy/download results or start over

**Important:** Guide users step by step. No jargon. No confusion.

---

## 5. DESIGN & UX REQUIREMENTS

**Color Scheme:** Blue and white, professional

**Visual Style:** clean

**Tone:** professional

**Platform Focus:** both

**Design Principles:**
- Minimal, distraction-free interface
- Clear visual hierarchy
- Generous white space
- Readable typography (16px minimum for body text)
- High contrast for accessibility
- Mobile-responsive design
- No login or account required
- Fast load times

---

## 6. TECHNICAL REQUIREMENTS

- Use semantic HTML5
- Modern CSS (Flexbox/Grid for layout)
- Vanilla JavaScript or React (choose based on complexity)
- No unnecessary dependencies
- Works in all modern browsers
- Responsive breakpoints: mobile (< 768px), tablet (768-1024px), desktop (> 1024px)
- Keyboard accessible
- Fast performance (optimized assets, minimal reflows)

---

## 7. EDGE CASES & SAFEGUARDS

- Handle empty inputs gracefully
- Validate all user inputs before processing
- Show appropriate error messages (never generic "Error" messages)
- Prevent form resubmission
- Handle loading/processing states
- Provide fallback content if something fails
- Set sensible defaults where possible
- Make assumptions explicit when needed

---

## 8. WHAT TO BUILD (EXPLICIT DELIVERABLES)

Build a complete, single-page web application that includes:

1. **HTML file** with proper structure and semantic markup
2. **CSS file** with all styles (no inline styles)
3. **JavaScript file** with all functionality
4. Clear, prominent headline explaining the app's purpose
5. Input form with proper labels and validation
6. Submit button with clear call-to-action text
7. Results section with formatted output
8. Copy/download functionality for results
9. "Start over" or "Reset" button
10. Responsive layout that works on all devices

---

## 9. CONSTRAINTS & RULES

- This prompt is complete and self-contained
- Do not ask follow-up questions
- Make reasonable assumptions where details are not specified
- Prioritize clarity and usability over feature complexity
- The app must be fully functional, not a prototype
- No authentication or backend required (unless explicitly needed)
- Keep the interface clean and minimal
- Every feature must serve the core user goal

---

## 10. FINAL INSTRUCTION

Build this application now. Make it production-ready, beautiful, and functional.
Do not return partial code or placeholders. Complete the entire implementation.
```
